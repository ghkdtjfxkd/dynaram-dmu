About Me

생성형 AI와 아키텍처 설계를 통해 복잡한 문제를 해결하고, 비즈니스 생산성을 극대화하는데 집중합니다.
기술은 사용자의 일상에 자연스럽게 스며들어 긍정적인 경험을 제공해야 한다고 믿습니다.

특히, LLM의 할루시네이션(Hallucination) 문제를 MSA와 SDD(Spec-Driven-Development)의 결합을 통해 구조적으로 해결하는 데 강점이 있습니다. 확장 가능한 시스템 설계를 통해 AI 모델의 잠재력을 최대한 발휘하고, 빠르게 MVP를 개발하여 가치를 검증하는 것을 즐깁니다.

💰 Pricing Strategy

  • ✔ 초기 전략: API 및 서버 비용의 손익분기점에 근접한 저렴한 구독료 책정으로 진입 장벽 최소화 및 사용자 확보.
  • ✔ Freemium: 무료 플랜 사용자에게는 배너 광고를 노출하여 수익 헷징.
  • ✔ Lock-in: 저렴한 구독 유지 비용으로 높은 잔존율(Retention) 확보.

🚀 Technical Roadmap

  • ➜ Phase 1: 공공 데이터 API 활용 및 Batch 처리를 통한 온프레미스 데이터 최신화 시스템 구축.
  • ➜ Phase 2: 축적된 알람 및 교통 데이터를 기반으로 자체 동적 시간 계산 알고리즘 AI 모델 개발.
  • ➜ Phase 3: 완전한 온프레미스 환경 전환으로 외부 API 의존도 제거 및 비용 절감.

5 Core Tech & Architecture

🏗 MSA & Load Balancing

아침 러시아워(Rush Hour)에 집중되는 트래픽을 효율적으로 처리하기 위해 MSA를 도입했습니다. 트래픽 급증 시 동적으로 스케일 아웃이 가능하며, 로드밸런싱을 통해 시스템 부하를 분산시킵니다.

🤖 SDD & GenAI

SDD(Spec-Driven-Development)로 AI의 출력 포맷을 엄격히 정의하여 할루시네이션을 방지하고, 안정적인 TTS 브리핑 스크립트를 생성합니다.

⚡ Batch Processing

공공 데이터 API의 호출 제한을 극복하고 응답 속도를 높이기 위해, Batch 작업을 통해 주요 노선의 데이터를 주기적으로 수집하여 내부 DB를 최신화합니다.

🔄 SAGA Pattern (EDA)

Kafka 기반의 이벤트 구동 아키텍처(EDA)와 SAGA 패턴을 적용하여, 분산된 마이크로서비스 간의 데이터 일관성을 보장합니다.
